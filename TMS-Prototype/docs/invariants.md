# Invariants (What Must Not Change)

## Engineering Intent → Validated Specifications
- Every implementation must stem from a clear, evidence-backed goal.

## Protected Invariants
- Customer trust
- Correctness
- Security
- Data integrity
- Architectural intent
- Traceability

## Traceability
- Every design decision, code change, and test must be linked back to an intent, assumption, or requirement.

## Knowledge Capture
- Each completed task must produce a reusable artifact (e.g., ADR, test pattern, decision log).

## Technology-Agnosticism
- The HEOS layers must not assume a specific language or framework; the prototype uses only the Python standard library to illustrate this principle.
